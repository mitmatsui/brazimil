Capítulo 1

Visão Geral do Sistema

O Brazimil — Análise Econômica é um sistema integrado de gestão desenvolvido especificamente para a Brazimil Indústria de Tintas. Ele centraliza precificação, tributação ZFM, evolução de vendas, fluxo de caixa e saúde financeira em um único painel acessível por qualquer navegador, com senha de acesso.

O que o sistema entrega

  • Cálculo correto da Margem de Contribuição com CE 55% (ZFM Bens Finais)
  • Simulação de preços em tempo real para qualquer produto e qualquer região
  • Comparação de dois cenários comerciais lado a lado
  • Evolução semanal de vendas por produto e por cliente
  • Fluxo de caixa das próximas 4 semanas com aging de inadimplência
  • Indicadores financeiros de liquidez, rentabilidade e endividamento
  • Acesso online pelo cliente via GitHub Pages com senha

Capítulo 3

Fluxo de Trabalho Semanal

O sistema foi projetado para ser atualizado uma vez por semana com o mínimo de trabalho manual. O fluxo completo leva menos de 10 minutos.

1
Operador do Omie exporta o relatório de vendas
Omie → Relatórios → Vendas → Faturamento por Período → últimas 13 semanas → Exportar CSV
Salvar como: vendas_omie.csv na pasta do projeto
2
Financeiro exporta a carteira de boletos do Bradesco
Bradesco Net Empresa → Cobrança → Consulta de Títulos → Exportar
Salvar como: boletos_bradesco.csv na pasta do projeto
3
Rodar o omie_sync.py para normalizar os dados
cd ~/Documents/brazimil-dashboard python3 omie_sync.py
Gera automaticamente: vendas.json e fluxo_caixa.json
4
Atualizar os indicadores financeiros (se necessário)
Abra o dashboard, vá na aba Indicadores Financeiros e atualize os campos editáveis com os valores do mês: Ativo Circulante, Estoque, Passivo Circulante, Passivo Total, Ativo Total, Patrimônio Líquido e Lucro Líquido.
5
Rodar o gerar.py para gerar o dashboard final
python3 gerar.py open brazimil_analise_economica.html
6
Publicar online pelo GitHub Desktop
GitHub Desktop → alterações aparecem automaticamente → Summary: "Atualização semanal" → Commit to main → Push origin
Aguarde 2 minutos. O cliente já acessa pela URL do GitHub Pages.
✅ Atalho completo no Terminal:

python3 omie_sync.py && python3 gerar.py && open brazimil_analise_economica.html

Este comando único roda os dois scripts em sequência e abre o dashboard no Chrome.
Capítulo 4

As 7 Abas do Dashboard

Aba 1
Painel Geral
Visão completa do portfólio com drill-down por categoria → subcategoria → produto. KPIs de MC média, saudáveis, atenção e prejuízo. Gráfico donut de distribuição. Tabela ordenável por qualquer coluna. Filtros de região, comissão e frete.
Aba 2
Simulador de Preço
Selecione qualquer produto, use o slider ou digite o preço. Veja em tempo real: decomposição do custo (fábrica, frete, impostos, MC), demonstrativo linha a linha e alerta colorido indicando se o preço é saudável, em atenção ou insuficiente.
Aba 3
Comparador de Cenários
Cenário A vs Cenário B com filtros independentes (região, comissão, frete). KPIs de cada cenário lado a lado. Top 20 produtos com maior diferença de MC em barras paralelas azul vs verde.
Aba 4
Tributação ZFM
Crédito Estímulo ajustável de 40% a 75%. Cálculo correto: ICMS 12% × (1−CE) − crédito insumos. Painel com todos os tributos (FMPES, UEA, FTI, PIS, COFINS). Comparativo CE correto vs modelo simplificado de 7%. Tabela ZFM por produto.
Aba 5
Evolução de Vendas
Dados do Omie via CSV. Faturamento semanal das últimas 13 semanas em barras + linha de tendência. Top 10 produtos e top 10 clientes por faturamento. KPIs: total do período, semana atual, variação % e média semanal.
Aba 6
Fluxo de Caixa
Dados do Bradesco via CSV. Projeção das próximas 4 semanas em barras empilhadas (prazo + vencido). Aging de inadimplência em donut (1–30, 31–60, 61–90, +90 dias). Top 8 inadimplentes. Lista detalhada de boletos com status colorido.
Aba 7
Indicadores Financeiros
Semáforo geral com 6 indicadores-chave. Grupo de Liquidez (corrente, seca, imediata, capital de giro). Grupo de Rentabilidade (margem bruta CALC, margem líquida, ROE, ROA, giro do ativo CALC). Grupo de Endividamento (grau, curto prazo, autonomia, inadimplência CALC, capital de terceiros). Gráficos históricos mensais para os 3 grupos. Campos editáveis para dados do balanço.
Capítulo 5

Indicadores Financeiros — Guia Completo

5.1 Como usar a aba

  1. Acesse a aba Indicadores Financeiros após o login
  2. Atualize os campos em azul mensalmente com os dados do balanço da Brazimil
  3. Os indicadores recalculam automaticamente ao digitar
  4. Os campos marcados com CALC são calculados automaticamente dos dados já existentes no dashboard
  5. Os campos marcados com edite precisam ser preenchidos manualmente

5.2 Grupo Liquidez — o que cada indicador significa

IndicadorFórmulaInterpretaçãoReferência Brazimil
Liquidez Corrente Ativo Circ. ÷ Passivo Circ. Capacidade de pagar dívidas de curto prazo com ativos de curto prazo ≥ 1,50 = Ótimo 1,00–1,49 = Atenção <1,0 = Crítico
Liquidez Seca (Ativo Circ. − Estoque) ÷ Passivo Circ. Igual à corrente mas exclui o estoque — mais conservadora ≥ 1,50 1,00–1,49 <1,0
Liquidez Imediata Caixa e Bancos ÷ Passivo Circ. Quanto a empresa consegue pagar imediatamente sem vender nada ≥ 0,20 0,10–0,19 <0,10
Capital de Giro Ativo Circ. − Passivo Circ. Folga financeira para financiar as operações do dia a dia Positivo = Ótimo Negativo = Crítico

5.3 Grupo Rentabilidade

IndicadorFórmulaInterpretaçãoReferência
Margem Bruta CALC MC % média do portfólio (ZFM CE 55%) Calculada automaticamente da aba Painel — já considera a tributação ZFM correta ≥ 35% 20–35% <20%
Margem Líquida Lucro Líquido ÷ Faturamento O que sobra de cada R$ vendido após todos os custos e impostos ≥ 10% 5–10% <5%
ROE Lucro Líquido ÷ Patrimônio Líquido Retorno que os sócios obtêm sobre o capital investido ≥ 15% 8–15% <8%
ROA Lucro Líquido ÷ Ativo Total Eficiência no uso de todos os ativos da empresa ≥ 8% 4–8% <4%
Giro do Ativo CALC Faturamento ÷ Ativo Total Quantas vezes o ativo se transforma em receita — eficiência operacional ≥ 1,0x 0,5–1,0x <0,5x

5.4 Grupo Endividamento

IndicadorFórmulaInterpretaçãoReferência
Grau de Endividamento Passivo Total ÷ Ativo Total Proporção do ativo financiada por terceiros — quanto menor, melhor ≤ 40% 40–60% >60%
Endiv. de Curto Prazo Passivo Circ. ÷ Passivo Total Concentração de dívidas com vencimento em até 12 meses — risco de liquidez ≤ 50% 50–70% >70%
Autonomia Financeira Patrimônio Líquido ÷ Ativo Total Quanto do ativo é financiado com capital próprio — inverso do endividamento ≥ 60% 40–60% <40%
Inadimplência CALC Vencido ÷ Carteira Total (Bradesco) Percentual de boletos vencidos e não pagos — puxado do fluxo de caixa ≤ 5% 5–15% >15%
Capital Terceiros/Próprio Passivo Total ÷ Patrimônio Líquido Para cada R$ próprio, quanto a empresa deve a terceiros ≤ 0,8x 0,8–1,5x >1,5x
💡 Dica de uso mensal: no primeiro dia útil de cada mês, abra o dashboard, vá em Indicadores Financeiros e atualize os 7 campos editáveis com os valores do balanço encerrado. Os gráficos históricos vão construindo o histórico mês a mês automaticamente durante a sessão.
Capítulo 6

Evolução de Vendas e Fluxo de Caixa

6.1 Como exportar do Omie

  1. Acesse o Omie com login da Brazimil
  2. Menu: Relatórios → Vendas → Faturamento por Período
  3. Selecione as últimas 13 semanas como período
  4. Clique em Exportar → CSV (separador ponto-e-vírgula)
  5. Salve como vendas_omie.csv na pasta brazimil-dashboard
Colunas esperadas do Omie: Data Emissão; Nº NF; Cliente; Produto; Qtd; Valor Unit; Valor Total. O omie_sync.py detecta automaticamente o separador e faz mapeamento flexível das colunas — pequenas variações no nome não causam erro.

6.2 Como exportar do Bradesco Net Empresa

  1. Acesse o Bradesco Net Empresa (não o app — usar o navegador)
  2. Menu: Cobrança → Consulta de Títulos
  3. Selecione todos os títulos (em aberto, vencidos e pagos recentes)
  4. Clique em Exportar → CSV
  5. Salve como boletos_bradesco.csv na pasta do projeto
Colunas esperadas do Bradesco: Nosso Nº; Cliente; Emissão; Vencimento; Valor; Status. O sistema detecta automaticamente "Pago", "Liquidado", "Vencido", "Em Aberto" e os classifica corretamente.

6.3 Modo exemplo automático

Se os CSVs não existirem na pasta, o omie_sync.py gera dados de exemplo realistas automaticamente para o dashboard funcionar visualmente. Um banner amarelo aparece no topo das abas avisando que são dados de exemplo. Substitua os CSVs pelos reais quando disponíveis.

Capítulo 7

Precificação e Tributação ZFM

7.1 A fórmula correta para Bens Finais com CE 55%

ComponenteFórmulaValor com CE 55%
ICMS débito saídaAlíquota interna AM12,0%
Crédito EstímuloCE × ICMS débito55% × 12% = 6,6%
Crédito insumos nacionaisAlíquota local AM7,0%
ICMS líquido a pagarMáx(débito − crédito entrada, 0)Máx(12%×45% − 7%, 0) = 5,4%
FMPES6% × ICMS débito × CE6% × 12% × 55% = 0,40%
UEAFixo sobre faturamento1,50%
FTI (insumo nacional)Fixo sobre faturamento1,00%
PISLucro Presumido — cumulativo0,65%
COFINSLucro Presumido — cumulativo3,00%
ComissãoDefinida nos filtros5,00% (padrão)
CARGA TOTALSoma de todos acima~16,95%
✅ Diferença crítica: a planilha anterior do time usava 7% de ICMS simples. Com CE 55% correto, o ICMS líquido é 5,4% — uma diferença de 1,6 pp que fazia a MC parecer 2 pp menor do que realmente é. O dashboard usa sempre a fórmula correta.

7.2 Cálculo da Margem de Contribuição

Frete = (peso_kg / 1000) × R$/ton Custo Var. = custo_fábrica + frete ICMS líq. = Máx(0,12 × (1 − CE) − 0,07, 0) FMPES = 0,06 × 0,12 × CE Carga total = ICMS_líq + FMPES + 0,015 + 0,01 + 0,0065 + 0,03 + comissão PLV = preço × (1 − carga_total) MC (R$) = PLV − Custo_Var MC (%) = MC ÷ PLV × 100 Preço mín. = Custo_Var ÷ ((1 − carga_total) × (1 − MC_alvo))

7.3 Classificação de status de MC

StatusCritérioCorAção recomendada
SaudávelMC% ≥ 35%VerdeProduto bem precificado — manter
Atenção20% ≤ MC% < 35%AmareloMonitorar — avaliar reajuste na próxima revisão
Crítico0% ≤ MC% < 20%Vermelho claroReajuste necessário — verificar custo e preço
PrejuízoMC% < 0%Vermelho escuroAção imediata — preço abaixo do custo variável
Capítulo 8

Ajustes via Python — Guia Prático

8.1 Mudar o preço de venda de um produto

No gerar.py, localize o dicionário tabela e encontre o produto. O preço de venda é o quarto valor da tupla:

# Formato: "NOME NO EXCEL": ("Categoria", "Subcategoria", "PRD_ou_None", preco) "BRAZIMIL EXTERNA 15L BALDE BRANCO NEVE": ("Tinta Base Água", "Externa", None, 99.90),

Se o produto tem um código PRD vinculado à planilha do time, o preço é puxado automaticamente de lá e o fallback é ignorado.

8.2 Adicionar um produto novo

Passo 1 — na seção tabela do gerar.py:

"BRAZIMIL NOVA LINHA 3L BALDE BN": ("Tinta Base Água", "Nova Linha", None, 45.00),
O nome entre aspas deve ser EXATAMENTE igual ao da aba 'Custo' do Excel — incluindo acentos, vírgulas e espaços. Copie direto do Excel para evitar erros.

8.3 Verificar por que um produto não aparece

Adicione este bloco temporariamente no gerar.py logo antes do print(f"Produtos:"):

# Depuração — remova depois de usar for nome_excel, (cat, sub, cod, preco_fb) in tabela.items(): custo = dados_custo.get(nome_excel) peso = dados_peso.get(nome_excel) if not custo or not peso: print(f" SEM DADO: {nome_excel[:55]}")

8.4 Criar uma nova categoria

Simplesmente use o nome novo na tabela. O dashboard cria o botão de filtro automaticamente:

"BRAZIMIL TEXTURA PREMIUM 18KG": ("Linha Premium", "Textura", None, 189.00),

8.5 Mudar a senha de acesso

No template.html, localize com Cmd+F:

var SENHA_CORRETA = "brazimil2026";

Mude para a nova senha, salve e rode python3 gerar.py.

8.6 Adicionar nova região tributária

No template.html, localize os 5 selects de região (um por aba) e adicione:

<option value="0.18">Bahia (18%)</option>

O value deve ser a carga total em decimal (ICMS + PIS + COFINS).

8.7 Mudar o piso de MC de 25% para outro valor

No template.html, dentro das funções calcZFM e calcMC:

// Mude 0.25 para o novo piso — ex: 0.30 para 30% var pmin = cv / ((1 - cargaTrib) * (1 - 0.25));

Capítulo 9

Publicar no GitHub Pages

9.1 Usando o GitHub Desktop (recomendado)

  1. Abra o GitHub Desktop
  2. Selecione o repositório brazimil no menu à esquerda
  3. As alterações no brazimil_analise_economica.html aparecem automaticamente na lista
  4. No campo Summary (canto inferior esquerdo), escreva uma descrição: Atualização semanal — semana 23
  5. Clique em Commit to main
  6. Clique em Push origin (botão azul no topo)
  7. Aguarde 2 minutos

9.2 URL de acesso para o cliente

https://mitmatsui.github.io/brazimil/brazimil_analise_economica.html
⚠️ Repositório deve ser Public: no plano gratuito do GitHub, o Pages só funciona com repositórios públicos. Como o dashboard tem senha, o conteúdo fica protegido pelo login mesmo com o código-fonte visível.

9.3 Usando o Terminal (alternativa)

cd ~/Documents/brazimil-dashboard git add brazimil_analise_economica.html git commit -m "Atualização semanal - $(date +%d/%m/%Y)" git push

9.4 Verificar que o Pages está ativo

  1. Acesse github.com/mitmatsui/brazimil
  2. Clique em Settings → Pages
  3. Confirme: Source = main, Folder = / (root)
  4. O status deve mostrar "Your site is published at..."
